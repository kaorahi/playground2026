証明数・反証数

探索木の各局面について「勝ちを証明する難しさ」と「負けを証明する難しさ」を数で表し、次に調べるべき場所を選ぶための考え方を学びます。

1. インタラクティブデモ

各葉のプルダウンを変更すると、即座に証明数・反証数が再計算されます。茶色の経路は、現在の値から見て「次に調べるべき候補」へ向かう経路です。赤字の pn / dn は、直前の操作で値が変化した箇所です。

現在の状態

?根の証明数 pn
?根の反証数 dn
まだ操作はありません。

2. 何を数えているのか

証明数と反証数は、ゲーム木や論理木で使われる評価値です。あるノードが「成り立つ」と証明するために必要な最小コストを証明数、逆に「成り立たない」と証明するために必要な最小コストを反証数と呼びます。

pn は proof number の略で、証明数を表します。dn は disproof number の略で、反証数を表します。

証明数 pn

このノードを勝ち、真、成功として確定するために、最低いくつの未確定部分を解決する必要があるかを表します。

反証数 dn

このノードを負け、偽、失敗として確定するために、最低いくつの未確定部分を解決する必要があるかを表します。

未確定の葉は、まだ調べていない候補です。最初は証明にも反証にも 1 個の調査が必要なので、通常は pn = 1、dn = 1 と置きます。

3. OR ノードと AND ノード

探索木では、手番や論理の種類に応じてノードを OR と AND に分けます。この教材では、OR ノードを角丸の長方形、AND ノードをひし形で表示します。

種類 証明するとき 反証するとき
OR 子のうち 1 つが証明されればよいので、子の pn の最小値 すべての子を反証する必要があるので、子の dn の合計
AND すべての子を証明する必要があるので、子の pn の合計 子のうち 1 つが反証されればよいので、子の dn の最小値
OR: pn = min(child.pn), dn = sum(child.dn)
AND: pn = sum(child.pn), dn = min(child.dn)

4. 葉の値

葉が勝ち・負け・未確定のどれかになったら、次のように置きます。

葉の状態 pn dn 意味
勝ち 0 ∞ すでに証明済みなので、証明コストは 0。反証は不可能。
負け ∞ 0 すでに反証済みなので、反証コストは 0。証明は不可能。
未確定 1 1 調べればどちらかに進む可能性がある。

5. 「次に調べる候補」の決め方

証明数探索では、根の結論を最も早く変えそうな葉を優先します。この教材では、茶色の辺が、その候補へ向かう経路を表します。

OR ノードでは

どれか 1 つの子が証明されればよいので、pn が最小の子を選びます。

AND ノードでは

どれか 1 つの子が反証されると全体を崩せるので、dn が最小の子を選びます。

同点の場合は左側の子を選びます。「葉 ○ の勝敗を決定」ボタンは、この茶色の経路の先にある未確定の葉をランダムに勝ちまたは負けへ確定します。

6. このデモの主要コード

下のコードが、証明数・反証数の再帰計算部分です。この HTML ファイル全体に、表示・操作・再計算のコードが含まれています。

function evaluate(node) {
  if (node.children.length === 0) {
    if (node.status === "win") return { pn: 0, dn: INF };
    if (node.status === "loss") return { pn: INF, dn: 0 };
    return { pn: 1, dn: 1 };
  }

  const values = node.children.map(id => evaluate(byId(id)));

  if (node.kind === "or") {
    return {
      pn: Math.min(...values.map(value => value.pn)),
      dn: safeSum(values.map(value => value.dn))
    };
  }

  return {
    pn: safeSum(values.map(value => value.pn)),
    dn: Math.min(...values.map(value => value.dn))
  };
}
