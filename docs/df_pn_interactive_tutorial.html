df-pn 入門

Depth-First Proof-Number Search を、pn/dn だけでなく、しきい値制御まで含めて理解する教材です。

1. df-pn の本体

df-pn は、Proof-Number Search を深さ優先化した手法です。単に pn/dn を計算するだけではありません。各再帰呼び出しに proof threshold と disproof threshold を渡し、その部分木をどこまで深掘りしてよいかを制御します。

ノード n には pn(n), dn(n) があり、さらに探索時だけ thpn(n), thdn(n) が渡されます。探索は次の条件で続きます。

pn(n) < thpn(n) かつ dn(n) < thdn(n)

どちらかが境界以上になった時点で、その部分木から戻ります。これにより、全木を保持する best-first 探索ではなく、現在の再帰スタックを中心にした depth-first 探索になります。

df-pn の df は「最有望葉へ降りる」だけでなく、「しきい値で止まりながら、同じ部分木に何度も入り直す」構造にあります。

2. pn/dn の更新規則

OR node

どれか 1 つの子を証明できれば親を証明できます。親を反証するには、すべての子の反証が必要です。

pn = min(child.pn) dn = sum(child.dn)

AND node

すべての子を証明できれば親を証明できます。親を反証するには、どれか 1 つの子の反証で足ります。

pn = sum(child.pn) dn = min(child.dn)

3. df-pn 固有のしきい値計算

親 p から最有望子 n に降りるとき、子 n へ渡す thpn/thdn を計算します。このとき「二番手」の値が重要です。二番手を超えてまで同じ子を深掘りする必要はない、という境界になります。

OR node p

pn が最小の子 n を選びます。s は pn が二番目に小さい子です。

child.thpn = min(parent.thpn, pn(s) + 1) child.thdn = parent.thdn - dn(parent) + dn(n)

AND node p

dn が最小の子 n を選びます。s は dn が二番目に小さい子です。

child.thpn = parent.thpn - pn(parent) + pn(n) child.thdn = min(parent.thdn, dn(s) + 1)
OR では証明競争なので二番手 pn が proof threshold を作ります。AND では反証競争なので二番手 dn が disproof threshold を作ります。

5. 疑似コード

dfpn(n, thpn, thdn):
    recompute pn(n), dn(n)

    while pn(n) < thpn and dn(n) < thdn:
        if n is terminal:
            set pn/dn from the result
            return

        if n is unexpanded:
            generate children
            initialize child pn/dn
            recompute pn(n), dn(n)
            if pn(n) >= thpn or dn(n) >= thdn:
                return

        if n is OR:
            n1 = child with smallest pn
            n2 = child with second smallest pn
            child_thpn = min(thpn, pn(n2) + 1)
            child_thdn = thdn - dn(n) + dn(n1)
        else:
            n1 = child with smallest dn
            n2 = child with second smallest dn
            child_thpn = thpn - pn(n) + pn(n1)
            child_thdn = min(thdn, dn(n2) + 1)

        old = (pn(n1), dn(n1))
        dfpn(n1, child_thpn, child_thdn)
        recompute pn(n), dn(n)

        if (pn(n1), dn(n1)) == old:
            return

実装では置換表、局面の合流、反復局面、手の生成順、コスト付き拡張などが加わります。ただし中核は、二番手の値を使って子の threshold を作り、threshold に達したら戻ることです。

6. よくある誤解

誤解修正
pn/dn を計算して小さい方へ行けば df-pnそれは PNS 的な説明です。df-pn では thpn/thdn が不可欠です。
深さ優先とは、ただ再帰で書くことしきい値により「ここまで調べれば十分」という停止条件を持つことが重要です。
二番手の子は探索しないので不要二番手の pn/dn は、最有望子へ渡す境界を決めるために必要です。
root で普通の反復深化をする標準的な df-pn では root の thpn/thdn を無限大に置き、境界を下位へ分配します。

7. 確認問題

OR node p で、親の thpn=10, thdn=20、子 n が最小 pn で pn=3,dn=5、二番手 s の pn=7、親の dn=12 です。n に渡す threshold はどれですか。
