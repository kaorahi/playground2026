df-pn インタラクティブ教材

Depth-First Proof-Number Search を、pn/dn だけでなく、しきい値制御まで含めて理解するための可視化です。

AND/OR tree proof number disproof number thresholds most-proving node depth-first

1. df-pn の本体

df-pn は、Proof-Number Search を深さ優先化した探索法です。単に pn/dn を計算して小さい方へ進むだけではなく、各再帰呼び出しに proof threshold と disproof threshold を渡し、その部分木をどこまで深掘りしてよいかを制御します。

ノード n には pn(n), dn(n) があり、探索中だけ thpn(n), thdn(n) が渡されます。探索は次の条件を満たす間だけ続きます。

pn(n) < thpn(n) かつ dn(n) < thdn(n)

どちらかが境界以上になった時点で、その部分木から戻ります。これにより、全木を広く保持する best-first 探索ではなく、現在の再帰スタックを中心にした depth-first 探索になります。

df-pn の df は「最有望葉へ降りる」だけではありません。「しきい値で止まりながら、同じ部分木を必要な範囲だけ掘る」構造にあります。

PNS との違い

項目素朴な PNSdf-pn
探索形best-firstdepth-first
主な制御全体から最有望葉を選ぶ最有望子 + しきい値で再帰する
メモリ探索木を広く保持しやすい再帰スタックと置換表を中心に保持しやすい
重要概念pn/dnpn/dn + thpn/thdn

2. pn/dn の更新規則

OR ノード

どれか 1 つの子を証明できれば親を証明できます。親を反証するには、すべての子を反証する必要があります。

pn = min(child.pn) dn = sum(child.dn)

AND ノード

すべての子を証明できれば親を証明できます。親を反証するには、どれか 1 つの子を反証できれば足ります。

pn = sum(child.pn) dn = min(child.dn)

3. 子へ渡すしきい値

親 p から最有望子 n に降りるとき、子 n へ渡す thpn/thdn を計算します。このとき二番手の値が重要です。二番手を超えてまで同じ子を深掘りする必要はない、という境界になります。

親が OR の場合

pn が最小の子 n を選びます。second_pn は pn が二番目に小さい子の値です。

child.thpn = min(parent.thpn, second_pn + 1) child.thdn = parent.thdn - parent.dn + child.dn

親が AND の場合

dn が最小の子 n を選びます。second_dn は dn が二番目に小さい子の値です。

child.thpn = parent.thpn - parent.pn + child.pn child.thdn = min(parent.thdn, second_dn + 1)

5. 疑似コード

dfpn(n, thpn, thdn):
    recompute pn(n), dn(n)

    while pn(n) < thpn and dn(n) < thdn:
        if n is terminal:
            set pn/dn from the result
            return

        if n is unexpanded:
            generate children
            initialize child pn/dn
            recompute pn(n), dn(n)
            if pn(n) >= thpn or dn(n) >= thdn:
                return

        if n is OR:
            n1 = child with smallest pn
            n2 = child with second smallest pn
            child_thpn = min(thpn, pn(n2) + 1)
            child_thdn = thdn - dn(n) + dn(n1)
        else:
            n1 = child with smallest dn
            n2 = child with second smallest dn
            child_thpn = thpn - pn(n) + pn(n1)
            child_thdn = min(thdn, dn(n2) + 1)

        old = (pn(n1), dn(n1))
        dfpn(n1, child_thpn, child_thdn)
        recompute pn(n), dn(n)

        if (pn(n1), dn(n1)) == old:
            # 置換表・循環・探索制限などで値が変わらない場合の無限ループ防止用の安全弁
            return

実装では置換表、局面の合流、反復局面、手の生成順、コスト付き拡張などが加わります。ただし中核は、二番手の値を使って子のしきい値を作り、しきい値に達したら戻ることです。

6. よくある誤解

誤解修正
pn/dn を計算して小さい方へ行けば df-pnそれは PNS 的な説明です。df-pn では thpn/thdn が不可欠です。
深さ優先とは、ただ再帰で書くことしきい値により「ここまで調べれば十分」という停止条件を持つことが重要です。
二番手の子は探索しないので不要二番手の pn/dn は、最有望子へ渡す境界を決めるために必要です。
